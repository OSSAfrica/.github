# Contributing to OSS-Africa

Thank you for your interest in contributing to Open Source & Security Africa (OSSAfrica)! We're excited to have you join our community.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion for improvement:

1. Check if an issue already exists for your concern
2. If not, create a new issue with a clear title and description
3. Include relevant details like steps to reproduce, expected behavior, and actual behavior

### Submitting Changes

1. Fork the repository
2. Create a new branch for your feature or fix (`git checkout -b feature/your-feature-name`)
3. Make your changes following our coding standards
4. Write clear, descriptive commit messages
5. Push your branch and submit a pull request
6. Link any related issues in your pull request description

### Pull Request Guidelines

- Keep pull requests focused on a single feature or fix
- Update documentation if your changes affect user-facing features
- Ensure your code follows the project's existing style
- Be responsive to feedback and make requested changes promptly

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please read it to understand the expectations for all community members.

## Getting Help

If you need help or have questions:

- Open a discussion in the repository
- Reach out to the maintainers

## Recognition

All contributors are valued members of our community. We appreciate every contribution, whether it's code, documentation, bug reports, or helping other community members.
